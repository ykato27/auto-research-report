## 1. プロダクト概要

製造業向けWebベーススキルマネジメントシステム（X1）。  
Excel・紙管理されていたスキルマップ・資格管理・育成計画をクラウドで一元管理し、製造業の人材育成を戦略的に推進するSaaS。

**ビジョン**: 「つくる人が、いきる世界へ」

### X1とは

X1はSkillnoteの新世代製品の総称。従来のスキルマップ管理に加え、人材サーチAI・個人カルテ・X1-OCRなどのAI機能群を含む。製造業の現場主導のスキル（＝力量）管理に特化している。

**メインメニュー構成**: 力量管理 / 個人力量 / 分析 / 人員計画 / データ管理 / 設定 / ガイド / Myページ（計8つ。「分析」は契約プラン、「人員計画」は権限により非表示の場合あり）

### 主要機能（X1）

- **力量管理（スキルマップ）**: 「スキル」「教育」「資格」の3タイプを管理。縦軸メンバー×横軸力量のマトリクスで習熟度を可視化。AND条件によるレベルアップ自動認定、充足状況の色分け表示
- **評価・アセスメント**: 自己評価・上長評価・多面評価。力量レベルパターン（0〜4段階等）を定義し、分析用数値を設定。評価履歴・推移グラフ
- **育成計画・進捗管理**: OJT・研修・資格取得の管理と追跡。教育訓練記録・目標シート・育成計画AI（開発中）
- **レポート・ダッシュボード**: 多能工化率・スキルカバレッジ率・スキルギャップの可視化。保有者数・充足状況・有効期限アラートのレポート機能
- **コンプライアンス管理**: ISO9001/IATF16949対応の力量管理要求事項への対応（資格更新期限アラート・有効期限の色強調表示等）
- **キャリア・業務経歴管理**: 職位ごとの必要力量と保有状況の対比。業務経歴の記録（経験年数・内容）と全社横断の業務経歴マップダッシュボード
- **個人カルテ（オプション）**: 力量マスタのスキル取得状況をレーダーチャートで表示。上司とメンバーが面談時に現状把握・目標すり合わせに使う
- **人材サーチAI（オプション）**: 力量・業務経歴データをAIが解析し、自然言語検索で最適人材を探索。標準モデル（力量・経歴マッチ）と拡張モデル（適性も含むAI判断）。A〜Eランクで評価
- **X1-OCR（開発中・2026年6月トライアル開始）**: 免状等をAI-OCRで読み取り力量を自動更新。AIプランとしてのアップセルを想定
- **スキル自動判定（PoC中）**: 作業データや業務記録からスキルレベルを自動評価する機能

## 2. ターゲット顧客

### 業種・規模
- **業種**: 製造業（食品、電機、自動車部品、金属加工、化学、印刷など）
- **主力ターゲット**: エンプラ（大手企業）
- **対象規模**: 従業員1,000名以上の製造部門・工場

### 担当部門・ペルソナ

| ペルソナ | 関心事 |
|----------|--------|
| 製造部・現場管理者 | 多能工化推進、ライン停止リスク低減、工程カバレッジ改善 |
| 人事・教育担当 | 育成効果の可視化、OJT管理、研修記録の一元化 |
| 品質管理部 | ISO/QMS対応、有資格者管理、監査対応 |
| 経営層・工場長 | 技能伝承問題の解消、人材配置の最適化、DX人材育成効果の定量化 |
| 設計・生産技術職の部門長 | 技術人材の可視化・育成・キャリア開発の根拠ある計画（技術者育成ソリューション） |

### 顧客が抱える主な課題
1. **技能伝承問題**: ベテランの退職でノウハウが消える。後継者育成計画が立てられない
2. **スキルの見える化不足**: 誰が何をできるかわからない。Excel管理では更新・集計が困難
3. **多能工化の遅れ**: 特定工程を1人しかこなせず、欠員・休暇時にラインが止まる
4. **育成効果の不可視化**: 研修・OJTの効果が測れない。費用対効果を経営に説明できない
5. **DXスキル管理の空白**: DX研修を実施しても「誰がどのスキルをどのレベルまで習得したか」が残らない

## 3. 現在の事業フォーカス（2026年）

### 人材サーチAI（リリース済み・継続改善中）
- X1に組み込まれたAI検索機能。蓄積されたスキルデータから最適な担当者・候補者を自動推薦
- 自然言語（「EV車両の構造設計をリードできるエンジニア」等）でセマンティックサーチが可能
- 目的：スキルデータの蓄積→活用サイクルを実現し、「入力する意義」を顧客に実感させる

### 技術者育成ソリューション（2026年4月リリース・展開中）
- **背景**: 既存Skillnoteの提案は製造現場（技能者）向けであり、技術職（設計・開発・生産技術・プラントエンジニアリング）には刺さりにくかった
- **対象**: 技術職の部門長・課長クラス（設計・開発・生産技術・生産管理・プラントエンジニアリング）
- **提供価値**: スキルデータをもとに技術人材を可視化し、育成・配置・キャリア開発を根拠をもって計画・実行できる
- **プロダクト構成**: X1既存機能 + 新規オプション（個人カルテ、キャリアマップ、人材ポートフォリオ、多面的な評価、人材サーチAI）
- **差別化**: 人事部門ではなく事業部門での育成・配置に特化。細かい粒度のスキル・経験管理、プロジェクト経験の蓄積、本人のキャリア自律支援
- **現状**: 個人カルテ・キャリアマップのPRリリース済み（2026年4月、日刊工業新聞掲載）。セミナー展開中
- **顧客例**: ダイハツ・村田製作所（試験部門）、マックス・JFEエンジ（設計）、本田技研・マツダ（生産技術）、シマノ

### スキル自動判定（PoC中）
- 作業データや業務記録からスキルレベルを自動評価する機能
- 東洋エンジニアリング・日立インプでPoC実施中

### SDP（スキルデータプラットフォーム）
- X1に蓄積されたスキルデータと、タレントマネジメント・人事システムのデータを統合し、データドリブンな人材配置を実現するサービス
- 提供価値：スキルベース人材配置、後継者計画の高度化、戦略的要員計画、育成投資最適化、ジョブアーキテクチャ自動生成
- ターゲット：エンプラ企業（X1導入済み、1,000名以上）

### AI Agent（開発・PoC設計フェーズ）
- **目標レベル**: Level 2（エージェントが成果まで担う）を当面の目標として社長室合意済み。成果課金（Level 3）は現時点では見通しなし
- **解決課題**: スキルデータはあるが、管理職のリソース不足で活用できていない状態の解消
- **MVP機能（一気通貫）**: ①チャット形式で組織課題を分析 → ②個人別スキルギャップと優先順位を提示 → ③育成計画たたき台を自動生成・一括登録
- **差分蓄積設計（重要）**: AIが出したサジェストと顧客が実際に選択した内容の「差分」を取得・蓄積する設計が方針。この蓄積が製造業横断のベンチマークデータになる（例：AIが3点評価 → 上司が2点に修正 → その差分を蓄積・学習）
- **BPO＋エージェントの両輪**: 育成がうまい組織には仕組みと文化がある。エージェントが担えるのは仕組み側のコスト削減が主。文化・ウェット部分は人が必要。BPO（CS・コンサル）もセットで設計
- **PoCパートナー**: 三菱ケミカル（管理職1名が部下約100名の育成計画を担当）、KOKUSAI ELECTRIC

## 5. 中長期戦略：Palantir型インフラ化

### 3層モート構造

- **Layer 1（構造的ロックイン）**: 確定証跡・有資格者承認フロー → 移行 = IATF審査リスクに転化（Veevaモデル）
- **Layer 2（認識論的ロックイン）**: 乖離率 × BPO知見 → 職種別テンプレートに結晶化（Palantirモデル）
- **Layer 3（ネットワーク強制）**: Tier1 CSR組み込み → Tier2/3強制採用 → 乖離率が業界標準へ（IMDSモデル）

### 乖離率の業界標準化（最重要の差別化軸）

乖離率とは「AIエージェント評価 vs 上司確定の差分」のデータ。100社・1000社分蓄積すると「日本製造業における溶接Level3の業界標準評価基準」が帰納的に確立される。このデータは確定承認ワークフローを持つSkillnoteのプロダクトを使い続けることでしか生成できない。将来的に認証機関がIATF改訂WGでSkillnoteの乖離率データを「実証的根拠」として引用する世界を目指す。

### ISO 9001:2026 強制購買イベント

ISO 9001:2026（2026年秋発行予定）が「組織知識管理」を監査可能な要件として明示化。IATF 16949:2027ではデジタルデータ整合性も加わる。全IATF認定工場で「知識管理体制の文書化証跡」が必要になり、紙台帳は即不適合。これは競合比較で選ばれる機会ではなく「強制購買イベント」。**2025〜2028年が決定的ウィンドウ。**

## 6. CSの方向性（AI Ready 組織）

### 基本方針：モート路線（品質・顧客成果が主目的）

AI ReadyなCS組織の主目的を「品質・モート（顧客成果と関係の深さで参入障壁を築く）」に置く。製造業の顧客は効率的なレポートで更新するのではなく、自社の現場・力量向上・技能伝承を理解する"人"を信頼して長く使い続ける。

### CSの役割再定義

「Skillnoteを設定する人」→「顧客のものづくり人材戦略を共に設計し、その実現をエージェントの力で確実に回す人」。日々の監視・報告・一次対応はエージェントが回す。人間に残るのは事業課題から力量戦略を設計、エージェント出力を判断して最終責任を持つ、経営層との信頼と合意の形成。

### CSヘルススコア（5ディメンション設計・実装中）

`HealthScore = 製品活用度(35%) + エンゲージメント(20%) + サポート健全性(20%) + 成果達成度(15%) + 関係性リスク(10%)`

Phase 1: Zendesk + 製品ログのみ（ルールベース）。Phase 2でamptalk感情分析統合。製造業固有の配慮：季節性補正（年度末・修繕期）、多拠点顧客は中央値で総合評価。
